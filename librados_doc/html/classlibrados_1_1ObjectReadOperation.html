void librados::ObjectReadOperation::cache_evict ( )

evict a clean cache tier object

This should be used in concert with OPERATION_IGNORE_CACHE to avoid triggering a promote on the OSD (that is then evicted).

void librados::ObjectReadOperation::cache_flush ( )

flush a cache tier object to backing tier; will block racing updates.

This should be used in concert with OPERATION_IGNORE_CACHE to avoid triggering a promotion.

void librados::ObjectReadOperation::cache_try_flush ( )

Flush a cache tier object to backing tier; will EAGAIN if we race with an update. Must be used with the SKIPRWLOCKS flag.

This should be used in concert with OPERATION_IGNORE_CACHE to avoid triggering a promotion.

list snapshot clones associated with a logical object

This will include a record for each version of the object, include the "HEAD" (which will have a cloneid of SNAP_HEAD). Each clone includes a vector of snap ids for which it is defined to exist.

NOTE: this operation must be submitted from an IoCtx with a read snapid of SNAP_DIR for reliable results.
