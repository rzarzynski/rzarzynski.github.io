175 /*
176  The manifest defines a set of rules for structuring the object parts.
177  There are a few terms to note:
178  - head: the head part of the object, which is the part that contains
179  the first chunk of data. An object might not have a head (as in the
180  case of multipart-part objects).
181  - stripe: data portion of a single rgw object that resides on a single
182  rados object.
183  - part: a collection of stripes that make a contiguous part of an
184  object. A regular object will only have one part (although might have
185  many stripes), a multipart object might have many parts. Each part
186  has a fixed stripe size, although the last stripe of a part might
187  be smaller than that. Consecutive parts may be merged if their stripe
188  value is the same.
189 */
