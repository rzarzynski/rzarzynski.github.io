186 /*
187  The manifest defines a set of rules for structuring the object parts.
188  There are a few terms to note:
189  - head: the head part of the object, which is the part that contains
190  the first chunk of data. An object might not have a head (as in the
191  case of multipart-part objects).
192  - stripe: data portion of a single rgw object that resides on a single
193  rados object.
194  - part: a collection of stripes that make a contiguous part of an
195  object. A regular object will only have one part (although might have
196  many stripes), a multipart object might have many parts. Each part
197  has a fixed stripe size, although the last stripe of a part might
198  be smaller than that. Consecutive parts may be merged if their stripe
199  value is the same.
200 */
